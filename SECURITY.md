# Security Policy

## Authorized use

TenantGuard is a defensive security tool for authorization regression testing.

Use it only against:

- applications you own,
- local development environments,
- staging systems you are allowed to test,
- client systems with written permission,
- explicitly authorized lab, demo, or CTF environments.

Do not use TenantGuard against third-party systems without clear written permission from the owner.

## Reporting vulnerabilities

If you discover a security issue in TenantGuard itself, please report it responsibly through GitHub Security Advisories or a private issue if advisories are not available.

## Project scope

TenantGuard is intentionally limited in scope. It is not designed for:

- brute-force attacks,
- credential stuffing,
- rate-limit bypass,
- stealth scanning,
- exploit delivery,
- unauthorized testing of third-party systems.

Features that expand the tool into offensive or unauthorized use are out of scope.
